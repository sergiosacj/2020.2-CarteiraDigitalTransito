    Cenários

    1. Introdução

      O propósito desse documento é exibir a metodologia utilizada para aplicar o método de cenários na modelagem dos requsitos e seus resultados.
      A técnica de cenários se trata de utilizar um conjunto de narrativas de situações e caminhos que se encontram no escopo do sistema e que possibilita a identificação de componentes de design. Em outras palavras, os cenários descrevem, narrando de forma textual, o sistema da perspectiva de um ou mais atores em determinadas situações de uso das funcionalidades.

    2. Metodologia

      A aplicação da técnica deu-se com auxílio dos participantes da equipe que utilizam a aplicação. É interessante mostrar que, por ter um escopo razoavelmente grande, o CDT tem uma gama muito ramificada de usuários por funcionalidades. Por exemplo, nem sempre um usuário tem veículos em seu nome e, se ele fosse utilizado para traçar os cenários, as funcionalidades referentes a veículos ficariam de fora da técnica.
      Sendo assim, foi utilizado também o tutorial de utilização do CDT disponibilizado pelo DENATRAN, para identificar o máximo de possibilidades de cenários.

    2.1. Nomenclatura

      Para especificar os cénarios foram utilizadas tabelas contendo os seguintes Elementos:

    • Objetivo: o que os atores tem como finalidade nesse cenário;
    • Contexto: onde se inicia a interação, quanto tempo dura, e qual o estado dos atores antes de iniciar;
    • Atores: todos os envolvidos que tem participação no cenário;
    • Recursos: o que é necessário para a situação acontecer;
    • Exceção: o que levará os atores para fluxos de exceção;
    • Episódios: narração detalhada das interações;
    • Rastreabilidade: os requisitos envolvidos.

    3. Cenários

      A seguir estão os resultados da aplicação dessa técnica. Os cenários a seguir contemplam todos os requisitos referentes as funcionalidades já presentes na aplicação.

    C01 - Cadastrar

    Elementos Respostas
    Objetivo Criar uma conta no GOV.br.
    Contexto - Local: Tela inicial da aplicação;
    - Tempo: 5 a 7 minutos;
    - Pré-condição: Acessoa à internet.
    Atores Usuários
    Recursos Smartphone, internet e CDT instalado.
    Exceção - Smartphone descarrega;
    - Perda de conexão.
    Episódios - Usuário abre a aplicação;
    - Usuário seleciona a opção entrar com GOV.br;
    - Usuário seleciona a opção "Crie sua conta";
    - Usuário seleciona uma opção de cadastro no sistema GOV.br;
    - Usuário informa os dados requisitados na opção selecionada.
    Rastreabilidade RF8

    C02 - Realizar o primeiro login na aplicação

    Elementos Respostas
    Objetivo Realizar login no CDT pela primeira vez.
    Contexto - Local: Tela inicial da aplicação;
    - Tempo: 1 a 2 minutos;
    - Pré-condições: Acesso à internet e cadastro no sistema GOV.br.
    Atores Usuário
    Recursos Smartphone, internet e CDT instalado.
    Exceção - Smartphone descarrega;
    - Perda de conexão;
    - Usuário não possuir cadastro no GOV.br;
    - Usuário esquece a senha;
    - Usuário não recebe o código de validação.
    Episódios - Usuário abre a aplicação;
    - Usuário seleciona a opção "Entrar com GOV.br";
    - Usuário informa o CPF e senha;
    - Usuário selecionaa opção "Entrar";
    - Usuário preenche o campo "celular";
    - Usuário seleciona a opção "Enviar Código.";
    - Usuário recebe o código de confirmação por sms;
    - Usuário digita o código recebido no campo "Código de confirmação SMS";
    - Usuário seleciona a opção "validar código";
    - Usuário autoriza utilização dos dados pessoais na aplicação.
    Rastreabilidade RF1

    C05 - Baixar a CNH digital com validação por certificação digital

    Elementos Respostas
    Objetivo Baixar o documento de CNH no APP.
    Contexto - Local: Aba Habilitação;
    - Tempo: 1 minuto;
    - Pré-condição: Estar logado no CDT, ter certificação digital ativa no portal do DENATRAN e ser o primeiro documento digital a ser baixado.
    Atores Usuário
    Recursos Smartphone, internet e CDT instalado.
    Exceção - Smartphone descarrega;
    - Perda de conexão;
    - Usuário não ter certificação digital do DENATRAN.
    Episódios - Usuário seleciona a opção "Habilitação" na Home do CDT;
    - Usuário seleciona a opção "Toque para baixar sua CNH";
    - Usuário seleciona a opção "Com certificado digital";
    - Usuário cadastra uma chave de acesso;
    - Usuário seleciona sim ou não para biometria em dispositivos compatíveis.
    Rastreabilidade RF3, RF10, RF19

    Elementos Respostas
    Objetivo Baixar o documento de CNH no APP.
    Contexto - Local: Aba Habilitação;
    - Tempo: 2 minutos;
    - Pré-condição: Estar logado no CDT e ser o primeiro documento digital a ser baixado.
    Atores Usuário
    Recursos Smartphone, internet, CDT instalado e QR code da CNH.
    Exceção - Smartphone descarrega;
    - Perda de conexão;
    - Usuário não ter o QR code da CNH;
    - Usuário não saber o CEP.
    Episódios - Usuário seleciona a opção "Habilitação" na Home do CDT;
    - Usuário seleciona a opção "Toque para baixar sua CNH";
    - Usuário seleciona a opção "Validar utilizando o celular";
    - Usuário concede acesso do CDT à câmera do aparelho;
    - Usuário avança nas informações deslizando para esquerda;
    - Usuário seleciona a opção "Vamos começar?"
    - Usuário escaneia o QR code da CNH;
    - Usuário seleciona a opção "Avançar";
    - Usuário escaneia o seu rosto de olhos fechados;
    - Usuário escaneia o seu rosto de olhos aberto e sorrindo;
    - Usuário informa o CEP do seu endereço;
    - Usuário seleciona a opção "Validar";
    - Usuário informa o número de telefone com DDD;
    - Usuário seleciona a opção "Concluir";
    - Usuário cadastra uma chave de acesso;
    - Usuário seleciona sim o não para biometria em dispositivos compatíveis.
    Rastreabilidade RF3, RF10, RF19
